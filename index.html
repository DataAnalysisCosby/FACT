    The FACT (Functions Are Classes Too) Programming Language

    Project maintained by rookieMP Hosted on GitHub Pages — Theme by mattgraham

    What is FACT?

    FACT is a new programming language that is designed to exploit the usefulness of mutable scopes (a.k.a. first class environments) to create a language that is both very simple and yet supports many complex paradigms (such as classes, inheritance, etc). FACT does not have any keywords such as "class" or "method," nor does it have special data types for objects (in the OOP sense that is). Indeed, classes and methods are easy to implement without such keywords, and there are only two data types: scopes and numbers (real numbers of arbitrary precision).

    What does the name "FACT" stand for?

    The name "FACT" stands for "Functions Are Classes Too," an allusion to on of the language's key features.

    What is FACT good for?

    While FACT is a Turing complete programming language and therefore theoretically could be used for a whole lot, the language is only in its infant stages, and so it really only excels as an arbitrary precision calculator. In the purely utilitarian sense right now it is essentially BC but nicer.

    Can you give me some sample code?

    Here are three ways to define a recursive factorial function in FACT:

    # Method one: 
scope fact1 @ (num n) 
{
  if (n > 1) 
    return n * fact1 (n - 1);
  return 1;
}
# Method two, using the "defunc" keyword:
defunc fact2 (num n) 
{
  if (n > 1) 
    return n * fact2 (n - 1);
  return 1;
} 
# Method three, using the "defunc" and "this" keyword:
defunc fact3 (num n) 
{
  if (n > 1) 
    return n * this (n - 1);
  return 1;
}

    FACT also has native support for threading. The following example, taken from the file thread_test.ft, illustrates how threading is done safely through message passing.

    # "${" runs its body in a new thread and returns the ID number of the new thread
num thread_1 = ${
  while (1) {
    scope m = receive ();
    if (m:message == "exit") {
      print ("exit signal received by thread ");
      print (m:sender);
      break;
    }
    print ('Message "');
    print (m:message);
    print ('" received from thread ');
    print (m:sender);
  }
  print ("Thread 1 exiting now.\n");
  send (0, "done");
};

send (thread_1, "Hello there!");
send (thread_1, "Im glad this works!");
send (thread_1, "now lets complicate things even further...");

num thread_2 = ${
  # Wait for a message of a thread id, and then send a message to that thread.
  while (1) {
    scope m = receive ();
    send (m:message, "exit");
    break;
  }
  print ("Thread 2 exiting now.\n");
};

send (thread_1, "are you ready thread 1? Actually I dont care!");
send (thread_2, thread_1);
send (thread_1, "and we're done!");

receive ();

print ("and that's the show folks!\n");
